1. **Run the helper first** to assemble the data-side JSON:

   Review the JSON. The helper already populated weather, calendar, mail,

2. **Read the spec** to confirm the executive-brief structure:
   - `goals/brief_saturday.md`
   - `~/Downloads/Weekend_Executive_Brief_Prompt.md` (the canonical spec)

3. **Enrich qualitative fields** the model is needed for. Use `patch`
   on `/tmp/sat-raw.json`. Apply the decision filter above to each:
   - `pillars.weather.whatToWear` — 1 sentence on today's forecast + tomorrow.
   - `pillars.life.oneThing` — low-friction Saturday action.
   - `pillars.life.rec` — 1 book/podcast/longread with blurb.
   - `pillars.vacation.driveDistanceIdeas` — filter to 2-4 season-relevant.
   - `pillars.retirement.planningNote` — Sat-tone forward planning nudge.
   - `pillars.executiveSummary.opportunities` — max 5 (high EV only).
   - `pillars.executiveSummary.summaryBullets` — 5 bullets, the spec's 5 angles.
   - `pillars.executiveSummary.actionItems` — max 5 ranked, with rationale + byWhen.
   - `pillars.executiveSummary.funFact` — 1, conversation-worthy.
   - `pillars.markets.whyParagraph` — 1 paragraph on why markets moved.
   - `pillars.investingThemes.themes` — Mag7 trends (filter `rawInputs.rss`).
   - `pillars.investingThemes.noMeaningfulNews` — true if nothing meaningful.
   - `pillars.retirementWatch.items` — what's new/actionable.
   - `pillars.aiLandscape.entries` — company one-liners from `rawInputs.rss`.
   - `pillars.health.entries` — 5 most important from `rawInputs.health.stories`.
   - `pillars.worthReading.articles` / `.videos` / `.podcasts` — 3 / 2 / 1.

4. **Write the enriched JSON** to `out/lifestyle/<date>.json`:
   ```
   cp /tmp/sat-raw.json out/lifestyle/<date>.json
   # then patch the qualitative fields with `patch` tool
   ```

5. **Ship it** — single helper call handles git push AND Vercel deploy:
   ```
   python3 scripts/build_lifestyle_json.py saturday --skip-deploy --date <date>
   ```
   `--skip-deploy` is critical here: the helper still does the manifest
   refresh + git commit + git push (the parts that finish in <2s), but
   skips the Vercel deploy (which can take 30-60s and risk the 3-min
   hard interrupt on cron sessions).

   After that returns 0, trigger the Vercel deploy in a separate fast
   background step:
   ```
   nohup bash -c 'cd /Volumes/BotCentral/Users/milo/repos/Milo/website && npm run fetch-briefs && vercel deploy --prod --yes' > /tmp/sat-deploy.log 2>&1 &
   ```
   `nohup ... &` detaches the deploy from the cron session so the 3-min
   hard interrupt on this prompt doesn't kill it mid-deploy. The Vercel
   build will keep running and finish on its own.

6. **Final response** — one-line confirmation:
   ```
   Published Saturday Lifestyle Brief for <date>: https://daily-brief-tau.vercel.app/weekend/<date>
   ```
   (The site may take ~60s to reflect the deploy; that's expected.)
